#### B1/B2/B3 — /qa slash command

Created `.claude/commands/qa.md` — invocable as `/qa` or `/qa deep`.

**Fast mode (`/qa`):**
1. Run `npx tsc --noEmit` and `npm run lint` — report pass/fail
2. Auth guard audit — read all `app/api/**/route.ts` files, verify each handler has auth check
3. Property scoping audit — verify all write operations have `.eq('property_id', PROPERTY_ID)`
4. Restricted zone compliance — cross-reference recent commits against restricted areas
5. Known-issue scan — Grep for `TODO`, `FIXME`, `HACK`, hardcoded values

**Deep mode (`/qa deep`):**
Fast checks, plus:
6. Onboarding flow trace — welcome → signup → confirm → setup-password → new-property
7. Invite flow trace — admin/invite → confirm → auth/join (verify no-op, pre-created membership)
8. Agent conversation flow trace — route.ts → handlers.ts tool dispatch, loop exits, error paths
9. Empty and error state coverage — each major UI tab
10. UX trust issues — loading states, destructive action confirmations, misleading copy

Both modes write a structured `PASS / WARN / FAIL` report to `QA_REPORTS/YYYY-MM-DD-[mode].md`.

**Estimated cost:** fast ~$0.15–$0.40, deep ~$0.75–$2.00. Deep takes 20–40 minutes.

Created `QA_REPORTS/` directory in-repo for durable report storage (B3).
