### Phase A1 — Defect triage and resolution

Fixed 6 defects from Erin and Dad's testing. All committed in one logical commit (9e71140).

**1. Agent recommending 2025 quarters**
- Root cause: system prompt had no awareness of the current date
- Fix: inject today's date, current year, and current quarter into system prompt on every request; added explicit instruction never to recommend past dates for new projects

**2. Visual tab showing Brady's layout to other users**
- Root cause: 3D scene in scene.tsx was hardcoded to 5090 Durham Rd with no property check
- Fix: added `isConfigured` prop to PropertyScene; scene shows an empty grid + "map not yet configured" message for any property whose address doesn't match a known configured layout; property.address is now passed from HomeDetailsTabs down to PropertyVisual

**3. Property dropdown not appearing for new users (Dad)**
- Root cause (probable): nested join `property_members → properties` in layout.tsx fails silently when RLS evaluation on the properties table blocks the nested select; property appears missing, propertyName is null, dropdown never renders
- Fix 1: split into two separate queries (membership IDs first, then properties by ID) to avoid nested RLS evaluation
- Fix 2: new-property/page.tsx now calls /api/property/switch before redirecting, so the parcel_property_id cookie is set on first load

**4. Floating chat and Agent tab were separate conversations**
- Root cause: each maintained independent local state initialized to a fresh greeting
- Fix: created components/agent-context.tsx with AgentProvider and useAgentContext hook; AppShell wraps both FloatingChat and page children with AgentProvider; both components share one message history; cascade messages (task completion flow) now append to the existing conversation instead of replacing it

**5. Cursor/project gap when dragging in quarterly scheduler**
- Root cause: DraggableCard was applying a CSS transform to move the original element AND DragOverlay was also rendering a moving clone — double visual with cursor offset
- Fix: removed transform from DraggableCard; original card goes to opacity 0 during drag; DragOverlay is the sole visual

**6. Agent responses too long and hard to read**
- Fix: system prompt now instructs 3–5 sentence replies, numbered lists only when needed, no padding or sign-offs; floating chat messages now render with whitespace-pre-wrap so line breaks display correctly
