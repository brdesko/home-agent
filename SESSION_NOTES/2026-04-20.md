## Decisions made

- **Auth: email/password instead of magic links.** Magic links require a working SMTP provider. Resend's `onboarding@resend.dev` address only works via their API, not SMTP. Gmail SMTP requires an app password which needs 2FA. Switched to email/password for simplicity; magic links can be revisited.
- **Next.js 16 `proxy.ts` instead of `middleware.ts`.** The scaffold installed Next.js 16.2, which deprecates the `middleware` file convention in favor of `proxy`.
- **Deleted `AGENTS.md`** generated by the scaffold — redundant with `CLAUDE.md`.

## Bugs fixed

- Self-referential RLS policy on `property_members` caused infinite recursion (`42P17`). Fixed by: (a) replacing the SELECT policy with a simple `user_id = auth.uid()` check, and (b) splitting the `FOR ALL` write policy into explicit INSERT/UPDATE/DELETE policies so the SELECT path is never recursive.
- `proxy.ts` was intercepting `/auth/callback` before the code exchange could run, sending users back to `/login`. Fixed by adding `/auth/` to the public route list.
